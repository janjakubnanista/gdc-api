register(userData, emailCredentials) → {Promise}

Register this user on server. After the user is succesfully registered, bootstrap data is fetched from server and stored in `data` property. You'll need to pass user options object as a parameter. This is basically a simplified user data object used to authenticate user. This object can contain following properties: - username (User's login) - password - firstName - lastName - phone - company In case the user options passed to this method (username and password) are not valid email credentials, you can pass these as a next parameter. Email credentails recognize two options, `username` and `password`. Returned promise is resolved with bootstrap data object.
Parameters:
Name Type Description
userData Object User data to use for registration
emailCredentials Object [optional] Email credentials to use when waiting for registration activation email
